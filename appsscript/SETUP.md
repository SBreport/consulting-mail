# 📤 Gmail 자동 발송 설정 가이드

웹앱의 "발송하기" 버튼으로 실제 메일이 나가게 하려면, 구글 시트에 발송 엔진(Apps Script)을 한 번만 설치하면 됩니다. **개발 지식 없이 따라 할 수 있게** 단계별로 정리했어요.

> 한 번만 설치하면 다음 기수(3기·4기)에도 그대로 재사용됩니다.

---

## 0. 준비물
- 발송에 쓸 **구글(Gmail) 계정** = 6월 설문 응답 시트를 소유한 계정 (보통 같은 계정)
- 시트에 **"발송완료" 열**이 있어야 함 (없으면 맨 오른쪽에 헤더 `발송완료` 추가)

## 1. 비밀 토큰 정하기
아무나 발송 엔진을 못 쓰게 막는 **비밀번호 같은 문자열**이에요. 아무거나 정하세요.
- 예: `smartbranding-2026-xyz`
- 이 값을 **두 곳(아래 3번 Code.gs, 그리고 config.js)에 똑같이** 넣을 거예요.

## 2. Apps Script 열기
1. **6월 설문 응답 구글 시트** 열기
2. 상단 메뉴 **확장 프로그램 → Apps Script** 클릭
3. 새 탭에 코드 편집기가 열림

## 3. 코드 붙여넣기
1. 편집기의 기존 내용(`function myFunction() {}` 등)을 **전부 지우기**
2. 이 폴더의 **`Code.gs` 내용 전체를 복사해서 붙여넣기**
3. 위쪽 `const TOKEN = 'PUT_YOUR_SECRET_TOKEN_HERE';` 에서 따옴표 안을 **1번에서 정한 토큰**으로 변경
   - 예: `const TOKEN = 'smartbranding-2026-xyz';`
4. 💾 저장 (Ctrl+S 또는 디스크 아이콘)

## 4. 발송 테스트 (배포 전 점검)
1. 코드에서 `testSend` 함수의 `TEST_EMAIL`을 **본인 이메일**로 변경
2. 편집기 상단 함수 드롭다운에서 **`testSend`** 선택 → **▶ 실행**
3. 처음 실행 시 **권한 승인 팝업** → 본인 계정 선택 → "고급" → "(안전하지 않음) 이동" → **허용**
   - (구글이 직접 만든 스크립트라 안전합니다. 본인 Gmail 발송·시트 수정 권한이에요.)
4. 본인 Gmail 받은편지함에 테스트 메일이 오면 성공 ✅

## 5. 웹앱으로 배포
1. 편집기 우측 상단 **배포 → 새 배포**
2. 톱니바퀴(유형 선택) → **웹 앱**
3. 설정:
   - 설명: 아무거나 (예: "컨설팅 메일 발송")
   - **다음 사용자로 실행: 나(본인 계정)**
   - **액세스 권한: 모든 사용자**
4. **배포** 클릭 → 나오는 **웹 앱 URL 복사** (`https://script.google.com/macros/s/.../exec` 형태)

## 6. 웹앱에 연결
복사한 URL과 토큰을 웹앱 설정에 넣어야 해요. **두 가지 방법** 중 편한 걸로:

**(A) 직접 입력** — `js/config.js`에서:
```js
dispatch: {
    gasUrl: '여기에_복사한_웹앱_URL_붙여넣기',
    token: '1번에서_정한_토큰'   // Code.gs의 TOKEN과 똑같이
}
```
수정 후 GitHub에 올리면 됨.

**(B) 저에게 전달** — URL과 토큰을 알려주시면 제가 config.js에 넣고 올려드릴게요.

## 7. 실제 발송
웹앱(사이트)에서 `Cmd+Shift+R`(강력 새로고침) 후:
- **개별 발송**: 응답자 선택 → "📤 발송하기"
- **일괄 발송**: "📤 미발송 전체 발송" (확인 창 → 진행)
- 발송 후 잠시 뒤 자동 새로고침 → 시트 "발송완료"에 `1`이 자동 기록되고, 앱에서 "발송완료"로 분류됨

## ⚠️ 알아두기
- **발송하면 바로 나갑니다** (검수 단계 없음). 처음엔 본인 이메일 행 하나로 테스트 권장.
- Gmail 하루 발송 한도: 개인 계정 **약 100통**. (한 기수 17명은 여유)
- 이미 발송완료(1/2/3)인 사람은 자동으로 **건너뜁니다** (중복 발송 방지).
- 보내는 주소·이름은 **이 스크립트를 만든 구글 계정 / "최형기"** 로 나갑니다.
